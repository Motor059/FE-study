## **JS을 이용하여 더 간결해진 HTML**
- 코드의 규모가 커질수록 HTML로 유지보수가 힘들어서 JS로 파일을 나눠서 관리하는 것이 효율적입니다.
- JS 파일들을 component로 이용하여, 즉 함수처럼 사용하여 재사용성을 올릴 수 있습니다.
- JS에 들어간 HTML을 넣어서 간단하고 직관적으로 표현하고, 이렇게 만들어진 JSX 문법을 BABEL이 자바스크립트 코드로 번역해줍니다.
- DOM이란 HTML 태그 하나하나를 자바스크립트로 컨트롤할 수 있도록 만든 객체 체계입니다. 

**React의 기초**
- vscode 내부 터미널에 `npx create-react-app week3`를 입력하여 week3폴더를 만들고, npm을 통해서 react를 하기 위한 파일들을 함께 다운 받습니다. 
- 터미널에서 cd week3로 이동을 하고 npm start를 하여 react로 구성된 웹페이지를 실행합니다.
- index.html과 index.js 내부 코드를 보며 root 태그를 이용하여 js코드를 컴포넌트로 활용한다는 것을 알았고, App.js를 수정하여 웹페이지 내용을 바꿀 수 있는 것을 알았습니다.

**컴포넌트 내에 사용한 주요 기능**
- `function` : JavaScript 함수를 선언하는 방식으로 react 컴포넌트를 만드는 기본적인 방식
    - `ex) function My_Info(){}`
- `<div>` : 구역을 나누는 기본적인 태그로 `<h1>, <p>`를 하나로 묶어주는 역할을 수행
- `import` : 가져오기 기능으로 다른 export한 javascript의 함수나 컴포넌트를 가져옴
    - `ex) import My_Info from './my_page/My_Info';`
- `export` : 내보내기 기능으로 다른 파일에서 import할 수 있도록 허용
    - `ex) export default My_Info;`

## **3주차 과제 수행**
- 기존 스터디에서 진행했던 내용처럼 하나의 파일에 작성될 수 있는 코드를 컴포넌트로 나누어서 자기소개 페이지를 구현했습니다.
- 저의 소개, 가족 소개, 대학 생활 소개, 최근 관심사 총 4가지의 컴포넌트로 나누어서 구성하였습니다.
- 여기에 List와 map 함수를 이용하여 App.js 내부에 들어갈 컴포넌트를 간략하게 표현해보았습니다.
- 처음에 map함수를 이용해서 return할 때 그냥 <Key />를 해주면 되는게 아닌가 싶어서 해봤고 됐습니다. 
- 하지만 리스트가 변하면 리스트 전체를 다시 그리는 비효율적인 점과 엉뚱한 컴포넌트와 연결될 수 있는 버그가 발생한다는 점을 보아 Key와 index를 필수적으로 이어주어야한다는 것을 알게 되었습니다. 

**JavaScript 문법 공부**
- `List` :  `[]`로 감싸져 있고 각 요소를 `,`로 구분하며 숫자, 문자열 등 어떤 자료형이든 담을 수 있습니다. 각 요소는 0부터 시작하는 인덱스를 가집니다.
    - `const page_list = [My_Info, My_Family, My_Campus, Dae_sang_hyeok];`

- `map 함수`: 배열의 모든 요소를 하나씩 꺼내서 새로운 값으로 바꾼 뒤 새로운 배열을 만듭니다. 원래 배열을 수정하지 않는 점이 중요합니다.
    - `const page = page_list.map((Key, index) => {return <Key k = {index}/>;});`
