## **React의 탄생**
- 페이스북을 예시로 좋아요를 눌렀을 때, 여러 파일을 request 및 response를 하면서 서버 비용이 많이 드는 문제가 발생합니다.
- DOM 조작을 이용해 좋아요 태그만 바꿔주면서 서버 비용을 많이 절감할 수 있습니다.
- 이렇게 태그들을 따로 관리하고 효율적인 관리를 위해 컴포넌트화한 다음에 개별적으로 작업할 수 있게 만든 것이 React입니다.

## **Virtual Dom의 탄생**
- DOM 조작은 브라우저 리소스가 많이 들어서 메모리 상에 가벼운 가상의 DOM을 미리 조작하고 결과만 모아서 Real DOM에 반영합니다.

**구현에 사용한 주요 기능**
- `useState` : React가 상태를 탐지할 수 있는 Hook으로, 바뀐 값이 화면에 반영되도록 re-render하는 역할을 수행합니다.
    - `ex) const [count, setCount] = useState(0)`

## **4주차 과제 수행**
- 스터디에서 진행했던 내용 중 counterNumber를 표기하는 컴포넌트를 따로 만든 것을 이용하여 버튼 또한 컴포넌트로 구현해보았습니다.
- 2주차에 진행했던 html로 구현한 카운터는 개발자가 직접 화면 업데이트에 대해 조작을 해야했습니다.
- 하지만 React로 구현한 카운터는 useState를 이용해 React가 상태 변화를 자동으로 감지하여 알아서 화면을 업데이트해주는 차이를 직접 느낄 수 있었습니다.
- 카운터에는 HTML이 더 쉬워 보이지만, APP이 복잡해질수록 버튼을 눌렀을 때 어느 화면을 바꿔야 할지에 대한 고민이 없어진 것이 장점으로 다가왔습니다.
