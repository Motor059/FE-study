## **서버와 통신하기**
- 누군가가 만들어놓은 서버인 JSON Placeholder에서 fetch와 then을 이용하여 todos를 받아와서 실습을 진행했습니다.
- 클라이언트가 서버에게 Todo 데이터를 요청하면 서버가 클라이언트에게 Todo를 보내줬다고 응답을 해줍니다.
- API는 클라이언트와 서버 간의 데이터를 주고 받는 통로 역할로 서로 다른 소프트웨어인 두 역할간 상호작용을 돕는 인터페이스입니다.

---

## **컴포넌트의 생애**
- Mount, Update, UnMount 총 세 단계로 구분되는데 새로운 props나 state를 수신하면 컴포넌트가 업데이트됩니다.
- fetch와 then만 이용하면 렌더링 무한 루프에 빠지게 되는데 useEffect와 빈 배열을 이용하여 무한 API 루프 문제를 해결할 수 있습니다. (한 번만 동기화)

**구현에 사용한 주요 기능**
- `fetch` : 자바스크립트에게 해당 주소로 가서 데이터를 가져오라고 명령합니다.
- `then` : response와 .json을 통해 해당 데이터를 쓸 수 있게 정리합니다.
- `useEffect` : React는 화면을 이쁘게 만드는 것이 주된 일이라서 서버에서 데이터를 가져오는 외부의 효과를 분리하기 위해 존재합니다.
    - `ex: useEffect(() => {수행할 코드}, [의존성 배열]);`
- `[의존성 배열]` : 없을 때(렌더링될 때마다 실행), 빈 배열(Mount 때 한 번만 실행), 값이 있는 배열(특정 데이터가 변할 때만 실행)

## **컴포넌트로 리팩토링**
- 기존 코드를 여러 파일로 분리하는 작업으로 App,js의 코드를 간소화시킵니다.
- 분리하는 과정 속에 계속 코드를 작성하는 것이 힘들어서 확장에서 Simple React Snippets를 설치하고 rfce같은 단축어를 자주 사용합니다.
- 이처럼 리팩토링은 결과의 변경 없이 코드의 구조를 재조정하는 작업입니다.
